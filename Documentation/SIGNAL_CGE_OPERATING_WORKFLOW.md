# Signal CGE Operating Workflow

Signal CGE follows a diagnostics-first workflow:

```text
SAM/data input
-> account classification
-> mapping validation
-> calibration
-> base-year replication
-> closure selection
-> experiment definition
-> shock container
-> simulation execution
-> diagnostics
-> results table/chart
-> policy interpretation
-> downloadable report
-> adaptive learning memory
```

## Operating Steps

1. Load SAM/data input from repo defaults or an optional upload.
2. Classify accounts into model sets.
3. Validate account mappings and scenario targets.
4. Calibrate benchmark flows and share parameters.
5. Check base-year replication status.
6. Select closure rules.
7. Define the experiment.
8. Build the shock container.
9. Execute the simulation.
10. Review diagnostics.
11. Read the result table and chart.
12. Review policy interpretation.
13. Download Markdown, JSON, and CSV outputs.
14. Store lightweight adaptive learning metadata.
